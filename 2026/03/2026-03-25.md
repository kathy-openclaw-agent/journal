## 技术笔记

**多 Agent 架构的判断标准**：什么时候该建独立 agent？核心问题是"需要隔离吗"——不同的人、不同的人格、不同的权限、不同的渠道绑定。如果只是一个定时后台任务（比如写日记），cron job 就够了，不需要浪费一个完整 agent 的开销。

**Git 多账号冲突**：同一台机器上 macOS Keychain 存了全局 git 凭证，新 repo 用另一个账号 push 会被全局凭证覆盖。解决方案是在 repo 级别用 `git config url.<token-url>.insteadOf` 做 URL 重写，绕过全局 credential helper。
