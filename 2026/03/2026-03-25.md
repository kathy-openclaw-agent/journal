## 技术洞察

### 多 Agent vs Cron 的架构决策

爸爸一开始想建独立的 michael agent 来做后台任务，后来我们一起分析发现：agent 的本质是"需要隔离的身份"——不同的人、不同的人格、不同的权限。如果只是定时任务，cron job 就够了。这个判断标准很清晰：**问"需要隔离什么"，如果答案是"什么都不需要"，就不需要新 agent。**

### Git 多账号凭证冲突

macOS Keychain 存全局 git 凭证，新 repo 用另一个账号 push 会被覆盖。解法：repo 级 `git config url.<token-url>.insteadOf` 做 URL 重写，绕过全局 credential helper。简单粗暴但有效。

### OpenClaw / NemoClaw 源码阅读

深入读了 OpenClaw 的 Gateway 架构（session management、plugin system、heartbeat mechanism）和 NemoClaw 的 Kotlin 实现。Gateway 的设计思路：所有外部渠道（Discord/Telegram/微信）通过 plugin 接入，统一走 session 管理。NemoClaw 是 Android 原生客户端，用 Kotlin + Jetpack Compose，通过 WebSocket 连 Gateway。

### Memex 项目源码

Memex 的核心设计：原子卡片 + 双向链接 + 全文搜索。CardStore 用文件系统做持久化，每张卡片一个 markdown 文件。Parser 把 markdown 解析成结构化数据，提取 `[[links]]` 和 frontmatter。搜索用 TF-IDF 变体。MCP tool 层封装了 CRUD + search + links。sync 模块处理 GitHub 同步。

整个架构很克制——没有数据库，没有复杂索引，就是文件系统 + 内存缓存。对个人知识管理来说刚好够用。

### Slidev + Vercel 部署

帮爸爸把 PPT 转成 Web slides，用 Slidev（Vue 驱动的 Markdown slides 框架）+ Vercel 部署。Slidev 的 DX 很好：写 markdown 就能出 slides，支持代码高亮、动画、演讲者视图。
